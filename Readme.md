
### Сервис укорачиватель ссылок


##### Стек

Golang, PostgreSQL, Docker


##### Цель

На обычную ссылку выдать укороченный URL со следующими параметрами:
-  длина 10 символов;
-  на один оригинальный URL должна ссылаться только одна сокращенная ссылка;
-  должна состоять из символов латинского алфавита в нижнем и верхнем регистре, цифр и символа _ (подчеркивание)


##### Запуск

К команде запуска предлагаются флаги:
-  `-repo` — для выбора хранилища (postgresql, inmemory)
-  `-config` — для выбора файла конфигурации (путь к нужному yaml-файлу)

В проекте представлен Makefile, где есть два сценария запуска:

1. Запуск через docker compose трех контейнеров: приложение, postgresql и миграции. Название команды: `startall` .
   
   Пример:
   
   ```sh
   make startall
   ```
   
   Для `docker compose down` можно просто прописать:
   
   ```sh
   make stopall
   ```

2. Запуск чисто Docker-образа приложения. Для корректного запуска нужно ВНИМАТЕЛЬНО проверить пути к файлу конфигурации и переменных окружения, также само содержимое файла конфигурации и переменных окружения.
   
   `buildapp` — собирает образ
   `runapp` — запускает контейнер на основе собранного образа
   `upapp` — собирает образ и запускает контейнер
   
   Можно также изменить параметры для запуска при необходимости (выбрать нужное хранилище, нужный порт и так далее).


##### Методы

Сервис предоставляет два метода:

1. Метод Post, который сохраняет оригинальный URL в базе и возвращает сокращённый

Синтаксис:

```sh
curl -X POST http://<host>:<port>/urls -d <request_body>
```

Пример:

```sh
curl -X POST http://localhost:8000/urls -d '{"url":"https://ozon.ru"}'
```

Ответ:

```sh
{"short_url":"hddqXuZ5Nn"}
```

2. Метод Get, который принимает сокращённый URL и возвращает оригинальный

Синтаксис:

```sh
curl -X GET http://<host>:<port>/{short}
```

Пример:

```sh
curl http://localhost:8000/urls/hddqXuZ5Nn
```

Ответ:

```sh
{"url":"https://ozon.ru"}
```
